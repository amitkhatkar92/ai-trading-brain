📈 WHAT GETS MEASURED
═══════════════════════════════════════════════════════════════════════════════

1. SIGNAL ACCURACY (Win Rate)
   ├─ Definition: Percentage of closed trades with positive P&L
   ├─ Goal: ≥ 40% (reasonable baseline for swing trading)
   ├─ Status Indicators:
   │  ├─ 40-50% = Good (average winning trades 2:1 ratio covers losses)
   │  ├─ 50-60% = Very good
   │  └─ 60%+ = Excellent
   └─ Data Source: paper_trades.csv (event: CLOSED, TARGET_HIT, STOPPED_OUT)

2. EXECUTION SLIPPAGE
   ├─ Definition: Average difference between entry price and actual execution
   ├─ Goal: < 0.15% per trade (minimal slippage)
   ├─ Status Indicators:
   │  ├─ < 0.15% = Excellent (minimal market impact)
   │  ├─ 0.15-0.25% = Good
   │  └─ > 0.25% = High (investigate liquidity)
   └─ Data Source: paper_trades.csv (entry_price vs target/stop_loss)

3. DRAWDOWN
   ├─ Definition: Current peak-to-trough decline in cumulative P&L
   ├─ Goal: < 5% of capital (controlled losses)
   ├─ Status Indicators:
   │  ├─ < 2% = Excellent
   │  ├─ 2-5% = Good (acceptable risk)
   │  ├─ 5-10% = Concerning
   │  └─ > 10% = Critical
   └─ Data Source: paper_trades.csv (cumulative P&L tracking)

4. SYSTEM UPTIME
   ├─ Definition: Percentage of time system was operational (no crashes)
   ├─ Goal: 100% (no crashes during session)
   ├─ Status Indicators:
   │  ├─ 100% = Perfect
   │  ├─ 99%+ = Excellent (1 crash tolerated)
   │  └─ < 99% = Needs investigation
   └─ Data Source: first_month_monitoring.json (startup/crash events)
