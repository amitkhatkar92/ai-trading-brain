2. BUGS FOUND & FIXED
────────────────────────────────────────────────────────────────────────────────

  BUG 1 — CRITICAL  |  strategy_lab/strategy_generator_ai.py
  ─────────────────────────────────────────────────────────────
  File    : strategy_lab/strategy_generator_ai.py
  Method  : _best_evolved_variant()
  Symptom : 0/N signals assigned every cycle — NO trades EVER executed.
  Root cause:
    The method picked the variant with the HIGHEST min_rr regardless of the
    signal's actual risk:reward ratio.
    e.g. Breakout_Volume_RSI_HiVol needs min_rr=2.5 but signal delivers
    R:R=2.0 → variant always rejected → 0 assignments.
  Fix:
    Added  min_signal_rr: float = 0.0  parameter.
    Only variants where  params["min_rr"] <= min_signal_rr  are eligible.
    Then picks the highest-scored among those.

  BUG 2 — CRITICAL  |  strategy_lab/strategy_generator_ai.py
  ─────────────────────────────────────────────────────────────
  File    : strategy_lab/strategy_generator_ai.py
  Method  : _load_evolved_strategies()
  Symptom : JSON's explicit min_rr values silently overwritten by base
            strategy value — EDG_MACRO__78_EE0000 had min_rr=2.0 in JSON
            but loaded as 2.5 (inherited from Breakout_Volume base).
  Fix:
    Changed  base_params["min_rr"]
         to  params.get("min_rr") or base_params["min_rr"]
    JSON value takes priority.
  Verified:
    EDG_MACRO__78_EE0000  min_rr: 2.0  ✅
    EDG_COMPOS_92_EE0002  min_rr: 1.8  ✅

  BUG 3 — CRITICAL (same file, same cycle as Bug 1)
  ─────────────────────────────────────────────────────────────
  File    : strategy_lab/strategy_generator_ai.py
  Method  : _pick_strategy() and _assign()
  Fix:
    Both methods now extract  rr = signal.risk_reward_ratio
    and pass  min_signal_rr=rr  to every _best_evolved_variant() call.

  BUG 4 — MINOR  |  main.py
  ─────────────────────────────────────────────────────────────
  File    : main.py
  Symptom : Hardcoded date  "Date: 2026-03-10"  in startup banner.
  Fix:
    Changed to  _dt.now().strftime("%Y-%m-%d")  (dynamic).
    Also removed a shadowing  from datetime import datetime as _dt
    inside the --discover block that caused UnboundLocalError.

  BUG 5 — MINOR  |  execution_engine/order_manager.py
  ─────────────────────────────────────────────────────────────
  File    : execution_engine/order_manager.py
  Symptom : Paper mode relied on kiteconnect simply not being installed.
            No explicit PAPER_TRADING guard, no trade journal.
  Fix:
    Added  self._paper_mode = getattr(_cfg, "PAPER_TRADING", True)
    Sets   self._broker = None  directly in paper mode — never loads broker.
    Logs   "[OrderManager] PAPER TRADING mode — no live orders will be sent."
    Added  _journal_write()  and  _journal_write_close()  methods.
    Appends every trade to  data/paper_trades.csv  (persistent across runs).
